O **Core Totem** é um sistema de totem de autoatendimento para restaurantes em servidores FiveM. Totens físicos são posicionados nos estabelecimentos, permitindo que jogadores comprem itens de forma autônoma — sem depender de funcionários atendendo. Cada restaurante tem seu próprio totem com estoque, preços e receita independentes.

## Sistema de Estoque e Preços

Cada restaurante possui uma lista de itens com estoque e preço gerenciáveis pelo admin. O estoque é compartilhado entre todos que acessam aquele totem.

| Configuração          | Descrição                                                              |
|-----------------------|------------------------------------------------------------------------|
| `DefaultStock`        | Estoque inicial de cada item ao criar o restaurante no banco           |
| `MaxStock`            | Limite máximo de estoque por item                                      |
| `RequireItems`        | Se `true`, o admin precisa ter o item no inventário para reabastecer   |
| `PriceRange.Min/Max`  | Faixa de preço permitida ao alterar valores                            |

---

## Bloqueio por Funcionários

Quando jogadores com a `ServicePermission` do restaurante estão online e dentro do raio configurado (`Blocking.Radius`), o painel de compras é bloqueado para clientes e a mensagem configurada em `Blocking.BlockedMessage` é exibida. Admins do restaurante continuam acessando o painel normalmente.

<Note>
  O bloqueio é global — aplica-se a todos os totens igualmente. Para desativar o bloqueio em um restaurante específico, defina `ServicePermission = false` naquela entrada.
</Note>

| Sistema      | Dependência          | Descrição                                          |
|--------------|----------------------|----------------------------------------------------|
| `sleepless`  | `sleepless_interact` | Label flutuante com ícone e tecla de interação     |
| `ox_target`  | `ox_target`          | Zona esférica clicável com crosshair               |
| `marker`     | nenhuma              | Marcador 3D desenhado no chão com tecla configurável |

## Saque de Receita

O faturamento do restaurante é acumulado a cada compra. O administrador pode sacar o valor de duas formas:

- **Item de dinheiro** — define `Withdraw.Item = "dollars"` (ou o item correspondente): o valor é entregue como item no inventário do admin.
- **Depósito bancário** — define `Withdraw.Item = nil`: o valor é depositado diretamente na conta bancária via `Functions.giveBankMoney`.
