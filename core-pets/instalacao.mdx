## Dependências

Antes de instalar, certifique-se de que as seguintes dependências estão presentes e funcionando no seu servidor.

<CardGroup cols={2}>
  <Card title="vRP Framework" icon="server">
    O Core Pets utiliza o sistema de usuários, permissões e configurações do vRP.
  </Card>
  <Card title="oxmysql" icon="database">
    Responsável por toda a comunicação com o banco de dados MySQL/MariaDB.
  </Card>
</CardGroup>

<Warning>
  O script **não funciona** sem o vRP e o oxmysql devidamente configurados. Verifique se ambos estão na sua `server.cfg` antes de prosseguir.
</Warning>

## Banco de Dados

Execute os comandos SQL abaixo no seu banco de dados para criar as tabelas necessárias.

<Steps>
  <Step title="Tabela de instâncias">
    Armazena cada pelúcia individualmente com seu nível, XP, benefícios e status de equip.

    ```sql
    CREATE TABLE IF NOT EXISTS `core_pets_instances` (
      `id`            INT           NOT NULL AUTO_INCREMENT,
      `user_id`       INT           NOT NULL,
      `item_key`      VARCHAR(100)  NOT NULL,
      `level`         INT           NOT NULL DEFAULT 1,
      `xp`            INT           NOT NULL DEFAULT 0,
      `benefits_json` TEXT          DEFAULT NULL,
      `equipped`      TINYINT(1)    NOT NULL DEFAULT 0,
      `created_at`    TIMESTAMP     NOT NULL DEFAULT CURRENT_TIMESTAMP,
      PRIMARY KEY (`id`),
      KEY `idx_user_id`       (`user_id`),
      KEY `idx_user_item`     (`user_id`, `item_key`),
      KEY `idx_user_equipped` (`user_id`, `equipped`)
    );
    ```
  </Step>

  <Step title="Tabela de pelúcia em destaque">
    Armazena a pelúcia marcada como destaque por cada jogador.

    ```sql
    CREATE TABLE IF NOT EXISTS `core_pets_featured` (
      `user_id`  INT          NOT NULL,
      `item_key` VARCHAR(100) NOT NULL,
      PRIMARY KEY (`user_id`)
    );
    ```
  </Step>

  <Step title="Tabela legada (compatibilidade)">
    Mantida para migração de dados antigos. Não remova se você está atualizando de uma versão anterior.

    ```sql
    CREATE TABLE IF NOT EXISTS `core_pets_collections` (
      `id`      INT          NOT NULL AUTO_INCREMENT,
      `user_id` INT          NOT NULL,
      `item_key`VARCHAR(100) NOT NULL,
      `qty`     INT          NOT NULL DEFAULT 0,
      PRIMARY KEY (`id`)
    );
    ```
  </Step>
</Steps>

## Instalação do Recurso

<Steps>
  <Step title="Copie a pasta do recurso">
    Coloque a pasta `core-pets` dentro do diretório de recursos do seu servidor (ex: `resources/[core]/`).
  </Step>

  <Step title="Adicione à server.cfg">
    Certifique-se de que o recurso é iniciado **após** o vRP e o oxmysql.

    ```
    ensure oxmysql
    ensure vrp
    ensure core-pets
    ```
  </Step>

  <Step title="Execute o SQL">
    Rode os scripts SQL da seção acima no seu banco de dados antes de iniciar o recurso pela primeira vez.
  </Step>

  <Step title="Configure o recurso">
    Edite os arquivos na pasta `config/` conforme descrito na seção abaixo.
  </Step>

  <Step title="Inicie o servidor">
    Reinicie o servidor ou use `refresh` + `start core-pets` no console.
  </Step>
</Steps>

## Configuração

Todos os arquivos de configuração estão em `config/`.

<AccordionGroup>
  <Accordion title="config.lua — Configurações principais" icon="sliders">
    **Comandos e permissões**
    ```lua
    Pets.Command      = "pets"     -- Comando para abrir o painel do jogador
    Pets.AdminCommand = "petsadm"  -- Comando para abrir o painel de admin
    Pets.AdminPermission = "Admin" -- Permissão vRP necessária para o admin
    ```

    **Sistema de trade**
    ```lua
    Pets.Trade = {
        RequireProximity = true,   -- Exige proximidade para iniciar trade
        MaxDistance      = 25.0,   -- Distância máxima em metros
        RequestTimeout   = 5000,   -- Tempo para aceitar o convite (ms)
        CleanupInterval  = 300000, -- Intervalo de limpeza de trades pendentes (ms)
        MaxActiveTrades  = 100,    -- Máximo de trades simultâneos no servidor
        MaxItemsPerTrade = 50,     -- Máximo de itens por trade
    }
    ```

    **Sistema de fusão**
    ```lua
    Pets.Fusion = {
        BaseChance        = 60,  -- Chance base de sucesso (%)
        RarityJumpPenalty = 25,  -- Penalidade por cada raridade acima da base (%)
        SameRarityBonus   = 10,  -- Bônus quando ambas as pelúcias têm a mesma raridade (%)
        MaxChance         = 90,  -- Chance máxima possível (%)
        MinChance         = 1,   -- Chance mínima possível (%)
    }
    ```

    **Cor do tema**
    ```lua
    Pets.ThemeColor = "#0143bb"  -- Cor principal da interface (hex)
    ```
  </Accordion>

  <Accordion title="catalog.lua — Catálogo de pelúcias" icon="list">
    Define todas as pelúcias disponíveis no sistema. Cada item segue a estrutura:

    ```lua
    ["chave_unica"] = {
        type     = "collection",
        name     = "Nome Exibido",
        category = "Categoria",
        rarity   = "rare",          -- commom | uncommom | rare | epic | legendary
        icon     = "https://cdn.corenetwork.dev/...",
        prop = {
            model    = "nome_do_modelo",
            loop     = true,   -- Animação em loop
            moving   = true,   -- Anima ao caminhar
            positions = {
                {
                    id    = "na_mao",
                    label = "Na Mão",
                    bone  = 12844,
                    pos   = { x, y, z },
                    rot   = { x, y, z },
                    model = "nome_do_modelo_posicao"
                },
                -- ...
            }
        }
    }
    ```

    <Note>
      Cada posição dentro de `positions` representa um local diferente do corpo onde a pelúcia pode ser equipada (mão, ombro, cabeça, etc.).
    </Note>
  </Accordion>

  <Accordion title="webhooks.lua — Webhooks do Discord" icon="discord">
    Configure as URLs dos webhooks para cada tipo de evento auditado.

    ```lua
    webhookURLs = {
        give     = "https://discord.com/api/webhooks/...",  -- Item adicionado via admin
        remove   = "https://discord.com/api/webhooks/...",  -- Item removido via admin
        trade    = "https://discord.com/api/webhooks/...",  -- Trade concluído
        featured = "https://discord.com/api/webhooks/...",  -- Destaque alterado
        deposit  = "https://discord.com/api/webhooks/...",  -- Depósito no painel
        withdraw = "https://discord.com/api/webhooks/...",  -- Saque do painel
        fusion   = "https://discord.com/api/webhooks/...",  -- Resultado de fusão
    }
    ```

    Deixe qualquer URL como `""` para desativar aquele webhook específico.
  </Accordion>
</AccordionGroup>

## Migração de Dados Antigos

Se você está **atualizando** de uma versão anterior que usava o sistema de stacks (quantidade), o Core Pets detecta automaticamente os dados antigos na tabela `core_pets_collections` e os converte para instâncias individuais na primeira inicialização.

<Note>
  A migração é **idempotente** — ela roda apenas uma vez e não duplica dados. Mantenha a tabela `core_pets_collections` no banco de dados mesmo após a migração.
</Note>
