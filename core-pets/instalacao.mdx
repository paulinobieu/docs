## Dependências

O Core Pets possui **apenas uma dependência obrigatória**. A integração com o seu framework é feita através de um arquivo de adaptação separado — não existe acoplamento direto com nenhum framework específico.

<CardGroup cols={2}>
  <Card title="oxmysql" icon="database">
    Única dependência obrigatória. Responsável por toda a comunicação com o banco de dados MySQL/MariaDB.
  </Card>
  <Card title="Qualquer framework" icon="puzzle-piece">
    ESX, QBCore, vRP, ox_core ou customizado. A integração é feita adaptando `config/functions.lua`.
  </Card>
</CardGroup>

<Info>
  O sistema de notificações detecta automaticamente o **ox_lib** e o utiliza se disponível. Caso contrário, usa o evento de notificação padrão do servidor.
</Info>

## Banco de Dados

Execute os comandos SQL abaixo no seu banco de dados para criar as tabelas necessárias.

<Steps>
  <Step title="Tabela de instâncias">
    Armazena cada pelúcia individualmente com seu nível, XP, benefícios e status de equip.

    ```sql
    CREATE TABLE IF NOT EXISTS `core_pets_instances` (
      `id`            INT           NOT NULL AUTO_INCREMENT,
      `user_id`       INT           NOT NULL,
      `item_key`      VARCHAR(100)  NOT NULL,
      `level`         INT           NOT NULL DEFAULT 1,
      `xp`            INT           NOT NULL DEFAULT 0,
      `benefits_json` TEXT          DEFAULT NULL,
      `equipped`      TINYINT(1)    NOT NULL DEFAULT 0,
      `created_at`    TIMESTAMP     NOT NULL DEFAULT CURRENT_TIMESTAMP,
      PRIMARY KEY (`id`),
      KEY `idx_user_id`       (`user_id`),
      KEY `idx_user_item`     (`user_id`, `item_key`),
      KEY `idx_user_equipped` (`user_id`, `equipped`)
    ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
    ```
  </Step>

  <Step title="Tabela de pelúcia em destaque">
    Armazena a pelúcia marcada como destaque por cada jogador.

    ```sql
    CREATE TABLE IF NOT EXISTS `core_pets_featured` (
      `user_id`  INT          NOT NULL,
      `item_key` VARCHAR(100) DEFAULT NULL,
      PRIMARY KEY (`user_id`)
    ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
    ```
  </Step>

  <Step title="Tabela legada (compatibilidade)">
    Mantida para migração de dados de versões anteriores. Não remova se estiver atualizando.

    ```sql
    CREATE TABLE IF NOT EXISTS `core_pets_collections` (
      `id`       INT          NOT NULL AUTO_INCREMENT,
      `user_id`  INT          NOT NULL,
      `item_key` VARCHAR(100) NOT NULL,
      `qty`      INT          NOT NULL DEFAULT 0,
      UNIQUE KEY `unique_user_item` (`user_id`, `item_key`),
      KEY `idx_user_id` (`user_id`),
      PRIMARY KEY (`id`)
    ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
    ```
  </Step>
</Steps>

## Instalação do Recurso

<Steps>
  <Step title="Copie a pasta do recurso">
    Coloque a pasta `core-pets` dentro do diretório de recursos do seu servidor (ex: `resources/[core]/`).
  </Step>

  <Step title="Adapte o config/functions.lua">
    Este é o único arquivo que precisa ser editado para integrar ao seu framework. Veja a seção [Adaptação de Framework](#adaptação-de-framework) abaixo.
  </Step>

  <Step title="Execute o SQL">
    Rode os scripts SQL da seção acima no banco de dados antes de iniciar o recurso pela primeira vez.
  </Step>

  <Step title="Configure o recurso">
    Edite os arquivos em `config/` conforme descrito na seção [Configuração](#configuração).
  </Step>

  <Step title="Adicione à server.cfg">
    Certifique-se de iniciar o recurso **após** o oxmysql e o seu framework.

    ```
    ensure oxmysql
    ensure seu-framework
    ensure core-pets
    ```
  </Step>
</Steps>

## Adaptação de Framework

Toda a comunicação com o framework está isolada em `config/functions.lua`. Para integrar com o framework do seu servidor, implemente as seguintes funções nesse arquivo:

<AccordionGroup>
  <Accordion title="Identificação do jogador" icon="user">
    O script precisa de um identificador numérico único por jogador para associar dados no banco.

    ```lua
    -- Retorna o ID único do jogador a partir do source (server ID)
    function Functions.getUserId(source)
        -- Exemplo vRP:
        return vRP.Passport(source)

        -- Exemplo ESX:
        -- local xPlayer = ESX.GetPlayerFromId(source)
        -- return xPlayer and xPlayer.getIdentifier() or nil

        -- Exemplo QBCore:
        -- local Player = QBCore.Functions.GetPlayer(source)
        -- return Player and Player.PlayerData.citizenid or nil

        -- Exemplo standalone (usa o próprio source):
        -- return source
    end

    -- Retorna o source do jogador a partir do ID único
    function Functions.getUserSource(userId)
        -- Implemente conforme seu framework
    end

    -- Retorna o nome completo do jogador
    function Functions.getUserName(userId)
        -- Implemente conforme seu framework
    end
    ```
  </Accordion>

  <Accordion title="Permissões de admin" icon="shield">
    Controla quem pode acessar o painel administrativo (`/petsadm`).

    ```lua
    -- Retorna true se o jogador tiver a permissão informada
    function Functions.hasPermission(userId, permission)
        -- Exemplo vRP:
        return vRP.HasPermission(userId, permission)

        -- Exemplo ESX (por grupo):
        -- local xPlayer = ESX.GetPlayerFromId(Functions.getUserSource(userId))
        -- return xPlayer and xPlayer.getGroup() == permission or false

        -- Exemplo QBCore:
        -- local Player = QBCore.Functions.GetPlayer(Functions.getUserSource(userId))
        -- return Player and Player.PlayerData.permission == permission or false

        -- Exemplo standalone (ace permissions):
        -- return IsPlayerAceAllowed(tostring(Functions.getUserSource(userId)), permission)
    end
    ```

    <Tip>
      O valor de `permission` verificado é o definido em `Pets.AdminPermission` no `config/config.lua`. Ajuste-o para corresponder ao sistema de permissões do seu framework (ex: `"admin"`, `"superadmin"`, `"god"`).
    </Tip>
  </Accordion>

  <Accordion title="Inventário (opcional)" icon="box">
    Necessário apenas se você quiser integrar pelúcias ao sistema de itens do seu framework (ex: dar a pelúcia via item no inventário).

    ```lua

        --     xPlayer.removeInventoryItem(item, amount)
        --     return true
        -- end
        -- return false
    end

    -- Adiciona um item ao inventário do jogador
    function Functions.giveInventoryItem(userId, item, amount)
        -- Exemplo ESX:
        -- local xPlayer = ESX.GetPlayerFromId(Functions.getUserSource(userId))
        -- if xPlayer then xPlayer.addInventoryItem(item, amount) end
    end
    ```
  </Accordion>

  <Accordion title="Notificações" icon="bell">
    O Core Pets detecta automaticamente o **ox_lib** e o usa se disponível. Caso contrário, usa o evento abaixo.

    ```lua
    -- Evento de notificação disparado pelo client-side quando ox_lib não está presente
    -- Adapte o nome do evento para o seu framework
    local NOTIFY_EVENT = "Notify"   -- ex: "esx:showNotification", "QBCore:Notify"
    ```

    Como o script já tem fallback automático para ox_lib, na maioria dos casos **não é necessário alterar nada** nessa parte.
  </Accordion>
</AccordionGroup>

## Configuração

<AccordionGroup>
  <Accordion title="config/config.lua — Configurações principais" icon="sliders">
    **Comandos e permissões**
    ```lua
    Pets.Command         = "pets"     -- Abre o painel do jogador
    Pets.AdminCommand    = "petsadm"  -- Abre o painel de admin
    Pets.AdminPermission = "Admin"    -- Permissão necessária para admin
    ```

    **Trade**
    ```lua
    Pets.Trade = {
        RequireProximity = true,    -- Exige proximidade para iniciar
        MaxDistance      = 25.0,    -- Distância máxima (metros)
        RequestTimeout   = 5000,    -- Tempo para aceitar o convite (ms)
        CleanupInterval  = 300000,  -- Limpeza de trades pendentes (ms)
        MaxActiveTrades  = 100,
        MaxItemsPerTrade = 50,
    }
    ```

    **Fusão**
    ```lua
    Pets.Fusion = {
        BaseChance        = 60,  -- Chance base de sucesso (%)
        RarityJumpPenalty = 25,  -- Penalidade por raridade acima da base (%)
        SameRarityBonus   = 10,  -- Bônus se ambas têm a mesma raridade (%)
        MaxChance         = 90,
        MinChance         = 1,
    }
    ```

    **Tema**
    ```lua
    Pets.ThemeColor = "#0143bb"  -- Cor principal da interface (hex)
    ```
  </Accordion>

  <Accordion title="config/catalog.lua — Catálogo de pelúcias" icon="list">
    Define todas as pelúcias disponíveis. Estrutura de cada item:

    ```lua
    ["chave_unica"] = {
        type     = "collection",
        name     = "Nome Exibido",
        category = "Categoria",
        rarity   = "rare",   -- commom | uncommom | rare | epic | legendary
        icon     = "https://cdn.corenetwork.dev/...",
        prop = {
            model    = "nome_do_modelo",
            loop     = true,
            moving   = true,
            positions = {
                {
                    id    = "na_mao",
                    label = "Na Mão",
                    bone  = 12844,
                    pos   = { x, y, z },
                    rot   = { x, y, z },
                    model = "nome_do_modelo_posicao"
                },
            }
        }
    }
    ```
  </Accordion>

  <Accordion title="config/webhooks.lua — Webhooks do Discord" icon="discord">
    ```lua
    webhookURLs = {
        give     = "",  -- Item adicionado via admin
        remove   = "",  -- Item removido via admin
        trade    = "",  -- Trade concluído
        featured = "",  -- Destaque alterado
        deposit  = "",  -- Depósito no painel
        withdraw = "",  -- Saque do painel
        fusion   = "",  -- Resultado de fusão
    }
    ```
    Deixe qualquer URL como `""` para desativar aquele webhook.
  </Accordion>
</AccordionGroup>

## Migração de Dados Antigos

Se você está atualizando de uma versão anterior que usava stacks (quantidade), o Core Pets converte automaticamente os dados da tabela `core_pets_collections` para instâncias individuais na primeira inicialização.

<Note>
  A migração roda apenas uma vez e é idempotente — não duplica dados. Mantenha a tabela `core_pets_collections` no banco mesmo após a migração.
</Note>
